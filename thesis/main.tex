\section*{Context}
% Set the scene: Active Directory is the identity backbone of ~90% of Fortune 500
% enterprises. Credential-based and ACL-based attacks (pass-the-hash, DCSync,
% Kerberoasting) are among the most impactful vectors in modern breaches.
% Defenders and red teams alike rely on tools like BloodHound to map the attack
% surface — but analysis remains manual and non-scalable.
Active Directory is the identity backbone of major enterprises, tho hidden withing 
the inside network of the company, once the outside defenses are breached 
(web services, phishing, usb stick ...) the advarsary finds himself able to directly interacte and query
the Active Directory Domain Controller itself through ldpa, smb, and other protocols presenting the entierty 
of the company IT infrastructure data and services to the attacker. A vulnerable AD poses extreme levels 
of risk making the hardening the AD network environment against an extreme priority.


Credential-based and ACL-based attacks(pass-the-hash, DCSync, Kerberoasting)

to map the attack surface and identify the vulnerable nodes during an engagement.

\section*{Main Contributions}
\begin{itemize}
  \item A complete, modular heterogeneous graph pipeline from raw BloodHound/Certipy
        JSON to navigable \gls{pyg} \texttt{HeteroData} tensors, covering
        \gls{adcs}, \gls{rbcd}, GPLink, and \gls{sid} history edges.
  \item A \gls{hgt}-based navigation policy trained on \gls{goad} traces and
        evaluated on the entirely unseen INLANEFREIGHT enterprise environment,
        demonstrating generalisation from a 351-node lab to a 4\,000-node network.
  \item Empirical evidence that the learned policy recovers paths with lower
        step-cost than BloodHound's shortest-path baseline on the same graph.
\end{itemize}

\section{Terminal State Learning via Offline Reinforcement Learning}
\label{sec:limit-rl}

% Current design: DCSync terminal is a hardcoded symbolic predicate
%   (GetChanges + GetChangesAll on the target domain).
% Problem: the beam search does not internalise the goal — it navigates
%   greedily then trims at the first terminal hit. This produces the
%   noise-tail issue observed in Result 3.
%
% Proposed future direction: offline RL over labelled rollouts.
%   - Label DCSync nodes as terminal states in the training traces.
%   - Train a value head (critic) that assigns high value to nodes with
%     DCSync rights on the target domain.
%   - The policy head learns to steer toward high-value nodes intrinsically,
%     without a hardcoded predicate.
%   - Algorithms: conservative Q-learning (CQL) or implicit Q-learning (IQL)
%     are well-suited to offline (batch) settings where online interaction
%     with a live AD environment is not feasible.
%
% Short-term interim: absorbing terminal state inside the beam loop
%   (freeze hypothesis at first DCSync hit, stop expanding) —
%   implementable without retraining, addresses the noise-tail immediately.
